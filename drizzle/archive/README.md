# Archived Migration Files

These SQL files are **NOT part of the active migration chain** and must never be run against production.

## Why they exist

During the account migration (July 2026), the project was moved from one Manus account to another.
The migration history was reset, and the Drizzle journal (`meta/_journal.json`) was rewritten to
start from scratch with a new baseline.

## Files

### `0000_organic_screwball.sql`
**DANGEROUS — contains only DROP TABLE statements for 14 production tables.**
This was generated by Drizzle when it detected the old schema and tried to produce a "clean slate"
migration for the new account. It was applied once against an empty database during the migration
setup, then the real data was imported. It must never run against a database that has live data.

### `0000_glamorous_stephen_strange.sql` and `0001_clumsy_quasar.sql`
These are the original migrations from the previous Manus account. They are superseded by the
current journal which starts at `0000_organic_screwball` (applied to empty DB) and `0001_known_shocker`
(adds the `metaConversionEvents` table).

## Current active migration chain

The active migrations tracked by `meta/_journal.json` are:
- `0000_organic_screwball` — baseline schema (applied to empty DB during migration)
- `0001_known_shocker` — adds `metaConversionEvents` table

All other `.sql` files in the parent `drizzle/` directory (0002 through 0019) are the full
historical migration chain from the original account. They are preserved for reference but
Drizzle will not re-apply them since they are not in the journal.
